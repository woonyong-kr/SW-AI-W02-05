# Week 5 - 핵심 역량 목표 설정

> 📅 작성 시점: 주차 시작 시

이번 주 과제 특성에 맞춰 10가지 핵심 역량별 목표를 구체적으로 설정합니다.
**달성률 산정을 위해 목표는 수치화 가능한 형태로 작성합니다.**

---

## 1. 문제해결
- **목표:** basic 과제 + 난이도 하 4개 + 난이도 중 6개 = 총 **10개 이상 과제 해결**
- **달성률 산식:** (해결한 과제 수 / 10) × 100


## 2. 설계
- **목표:** 수요 코딩회 React 핵심 기능 구현 시 **Component 구조, 루트 State 관리, Hooks 동작 흐름, Virtual DOM + Diff + Patch 연결 방식** 등 설계 포인트 **4개 이상**을 README 또는 설계 메모에 명시
- **달성률 산식:** (문서에 명시한 설계 포인트 수 / 4) × 100


## 3. 구현
- **목표:** basic 테스트 통과 + 난이도 하 4개 AC + 난이도 중 6개 AC + 수요 코딩회 핵심 기능 4개(Component, State, Hooks, 화면 동작 페이지) 구현 = **14개 이상 구현 완료**
- **달성률 산식:** (테스트 통과 + AC 완료 + 핵심 기능 구현 항목 수 / 14) × 100


## 4. 품질
- **목표:** 난이도 하·중 10개 백준 AC 통과 + 수요 코딩회 단위 테스트/기능 테스트 작성 및 통과 = **11개 항목**
- **달성률 산식:** (통과한 항목 수 / 11) × 100


## 5. 유지보수
- **목표:** 이번 주(금~목) **7일 중 5일 이상** 개인 레포지토리에 의미 있는 커밋 메시지로 알고리즘 풀이 및 학습 내용을 커밋·푸시
- **달성률 산식:** (커밋한 일수 / 5) × 100 (100% 초과 시 100%로 계산)


## 6. 협업
- **목표:** 팀 협업 룰/팁 합의(1개) + 수요 코딩회 팀 프로젝트 참여(1개) + 코어타임 코딩테스트 3문제 이상 팀 풀이(1개) = **3개 완료**
- **달성률 산식:** (완료한 항목 수 / 3) × 100


## 7. 태도
- **목표:** 공통 6개(목표 수립, 달성률 평가, AI 원칙, 업무 검토, WIL, 팀 협업 룰) 중 **6개 완료**
- **달성률 산식:** (완료한 공통 과제 수 / 6) × 100


## 8. 비즈니스 이해
- **목표:** 이번 주에 구현한 React-like 시스템이 **왜 필요한지**, 그리고 실제 React와 어떤 차이가 있는지를 README에 **2문장 이상** 명시
- **달성률 산식:** (명시한 문장 수 / 2) × 100


## 9. AI 활용
- **목표:** AI 활용 원칙 수립(1개) + 수요 코딩회 구현 코드의 핵심 로직 검증·원리 설명(1개) = **2개 완료**
- **달성률 산식:** (완료한 항목 수 / 2) × 100


## 10. 학습 민첩성
- **목표:** DP/Greedy 핵심 주제와 코어타임 문제 세트(`Climbing Stairs`, `House Robber`, `Word Break`, `Triangle`)를 포함해 **6개 이상 개념/문제 유형**을 학습 후 적용
- **달성률 산식:** (학습 후 적용 완료한 개념/문제 유형 수 / 6) × 100


---

### 달성률 산식 참고
- 문제해결/구현: 과제·문제·핵심 기능 단위로 카운트
- 품질: 백준 AC + 수요 코딩회 테스트 항목
- 유지보수: 일 단위 커밋 빈도
- 협업/태도/AI 활용: 완료 항목 수
- 비즈니스 이해: README 명시 문장 수 기준
