## 핵심 원칙

- AI는 학습 파트너이며, 결과보다 이해·검증·설명 가능 여부가 더 중요함
- 코어타임 문제 풀이 중에는 AI와 인터넷을 사용하지 않음
- 문제 풀이에서 AI를 사용할 때는 점화식, 상태 정의, 그리디 선택 근거를 직접 설명할 수 있어야 함
- 수요 코딩회에서는 AI를 적극 활용하되, 생성된 코드의 핵심 로직과 한계를 README 발표 수준으로 설명할 수 있어야 함
- `useState`, `useEffect`, `useMemo`, 루트 State 관리, Component 분리, Diff/Patch 과정은 AI 답변을 그대로 믿지 않고 직접 테스트와 예제로 검증하기
- AI가 생성한 코드는 바로 제출하거나 머지하지 않고, 테스트 케이스와 실제 동작 확인 후 사용하기
